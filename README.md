## How It Works

The script has basically two module: `file_utils` and `llm_utils`. `file_utils` handle the processing of source files, while
`llm_utils` handle the communication with LLM. `file_utils` use `pypdf` to split the source file into multiple chunks and stores
them in a directory named after the source file as `page_1.pdf...page-n.pdf`. This is necessary to ensure that the LLM's context window isn't exceeded. It also contains function to list all chunks in the directory.

`llm_utils` leverage Google's GenAI SDK to pass chunks to an LLM (Gemini 2.0 Flash, in this case) which functions as an OCR service. The responses are parsed and returned as Pydantic models. The final result is augmented with page numbers - since they are not available to the LLM due to only
the individual files being sense - and converted to JSON. The script requires a Google GenAI API key to be present as an enviroment
variable `GEMINI_API_KEY`

## Techonologies used

* Tradition file parser: PyPDF - to parse source file and chunk them
* LLM: Gemini 2.0 Flash - acts as an OCR service

## Limitations

The model tends to hallucinate occasionally when there is no content to be extracted from a chunk
